# 贡献指南

感谢您对 IGDM Universal 项目的关注！

## 🤝 如何贡献

### 报告 Bug

如果您发现了 Bug，请创建一个 Issue 并包含：
- Bug 的详细描述
- 重现步骤
- 预期行为
- 实际行为
- 屏幕截图（如果适用）
- 系统环境信息

### 建议新功能

我们欢迎新功能建议！请创建 Issue 并说明：
- 功能描述
- 使用场景
- 可能的实现方案

### 提交代码

1. Fork 项目
2. 创建功能分支 (`git checkout -b feature/AmazingFeature`)
3. 提交更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 创建 Pull Request

## 📝 开发规范

### 代码风格

- 使用 ESLint 检查代码
- 遵循 React 最佳实践
- 添加必要的注释
- 保持代码简洁清晰

### 提交信息

使用清晰的提交信息：
- `feat: 添加新功能`
- `fix: 修复 Bug`
- `docs: 更新文档`
- `style: 代码格式调整`
- `refactor: 重构代码`
- `test: 添加测试`

## 🧪 测试

在提交前请确保：
- 应用能正常启动
- 新功能按预期工作
- 没有破坏现有功能
- 代码通过 ESLint 检查

## 📄 许可证

通过贡献代码，您同意您的贡献将在 MIT 许可证下发布。
